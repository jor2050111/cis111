# CIS111 Textbook Style Guide

This file holds the CIS111-specific layer. The canonical shared writing
law lives in `docs/style-guide-core.md`. Read the core first. When this
file and the core differ, this file wins for CIS111 only.

## Audience and Tone Calibration

* **Assumed background:** No prior ethics or IT coursework.
* **Register:** Warm, direct, and professional. Treat students as capable
  adults who are learning a new reasoning discipline.
* **Reading level:** Flesch Reading Ease 60-70, leaning toward the upper
  end when legal or technical precision permits.
* **Bridging rule:** Define a concept before asking students to apply it.
  Use one short reminder when a later chapter returns to an earlier
  framework.
* **Example rule:** Use an IT decision with at least two defensible
  positions. Avoid a scenario whose answer is obvious from the setup.

## Bloom's-Level Emphasis for CIS111

* **Primary levels:** Understand, Apply, and Analyze in Parts I-II.
* **Advanced levels:** Apply, Analyze, and Evaluate in Part III.
* **Synthesis levels:** Analyze, Evaluate, and Create in Part IV.
* **Avoided verbs:** Never use `understand`, `know`, or `learn` as an MLO
  verb. The Bloom tag may be Understand when the measurable verb is
  explain, distinguish, summarize, or classify.
* **Distribution rule:** Each chapter has exactly three MLOs. They rise
  from the chapter's foundation to its Skills Lab judgment or creation.

## Tool Conventions

The course uses a modern web browser, Canvas LMS, and Microsoft Word
with PDF export.

* **Browser use:** Students open primary sources, confirm publication
  dates, and record source titles and URLs.
* **Canvas use:** Students download the CIS111 course pack and submit the
  finished document through the course assignment.
* **Word use:** Students use current Microsoft 365 Word desktop or Word
  for the web. Tasks must work in either unless a chapter states a clear
  desktop-only reason.
* **File format:** Students draft in `.docx` and submit PDF when the
  assignment requires a fixed layout.
* **Code fences:** Use `text` only when showing a short policy, prompt,
  or structured notation that students must copy exactly.

## Scenario and Course-Pack Conventions

* **Scenario domains:** Phoenix-area education, healthcare, local
  government, small business, software development, cybersecurity, and
  online platforms.
* **Provided files:** Each Skills Lab folder lives in
  `assets/code/chapter-NN/`. The folder contains a scenario or evidence
  packet plus a README that names the source and license.
* **Student access:** Students download one CIS111 course-pack archive
  from Canvas and keep the original folder structure.
* **Student work:** Students create their analysis in Word. They do not
  retype long evidence tables, policies, or source excerpts.
* **Source rule:** Use primary government, professional, court, or
  standards-body sources for legal and technical claims. Label
  fictional organizations and synthetic evidence.
* **Reproducibility:** Any generated evidence packet must state its
  assumptions and keep all numbers consistent with the chapter.

## Chapter Practice Pattern

The shared template calls the activity Predict-Run-Explain. CIS111 uses
the same three moves without software execution:

* **Predict:** Make an initial ethical judgment before seeing the full
  analysis.
* **Run:** Apply a named framework, rule, or stakeholder test.
* **Explain:** Defend the result with case evidence in one or two
  sentences.

Keep each practice block to 3-7 minutes. The activity should expose a
reasoning choice, not reward guessing.

## Skills Lab Convention

Every lab uses three progressive parts:

1. **Foundation:** Identify stakeholders, facts, duties, and the central
   ethical tension.
2. **Application:** Apply a framework, law, policy, or professional code.
3. **Extension:** Defend a recommendation for a named audience.

Students answer exactly two Questions & Analysis prompts. Every lab uses
the shared 16-point rubric without a chapter-specific rewrite.
