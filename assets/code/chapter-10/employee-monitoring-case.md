# TechBridge Monitoring Release Evidence

**Status:** Fictional instructional scenario

## Decision

TechBridge must choose a full launch, narrow pilot, or pause for a worker
monitoring tool.

## Business and Tool Evidence

* The firm has 200 workers who work at home and in the office.
* A client requires proof that source code is protected.
* Two people sent code to a personal cloud drive.
* The tool records screens and keys, takes webcam images every 10 minutes, and tracks firm-phone GPS at all times.
* The tool gives each worker a weekly output score.
* Three low scores trigger a job review.

## Vendor and Data Evidence

* The vendor will not show the weights used in its score.
* Its test set covers English-speaking office staff.
* It has no results for screen reader users, field staff, or staff who use more than one language.
* Raw data is kept for two years and can be exported by managers.
* The draft notice allows monitoring for any business need.
* The draft gives no appeal path.

## Speech and Law Evidence

* Fifteen workers signed a group letter about webcam use and pay.
* A manager wants to use the tool to identify the writers.
* The legal team has not reviewed federal interception law, state notice law, protected group activity, discrimination, contract, or EU scope.

## Student Task

Use the facts with the speech map, monitoring need test, data table, law
map, vendor audit, and release gate. State any assumption you add. Do not
treat a missing fact as proof.
